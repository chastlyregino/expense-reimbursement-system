Ticket
{
    id,
    user_id,
    amount,
    description,
    status,
    creation_timestamp,
    update_timestamp,
}

Once Login

As Employee
    - Request new tickets
    - View History of tickets

As Manager
    - View pending tickets
    - Update pending tickets


Ticket life flow

New ticket - as Employee
    - check if details are complete
    - add to database (status:pending)

View History - as Employee
    - user_id return all tickets (include creation timestamp)

View All - as Manager
    - return all tickets (FIFO)
    
    View Pending - as Manager
        - return all pending tickets (FIFO)

    Update Pending - as Manager
        - ticket_id update status (approved/denied)
        - return updated pending tickets
